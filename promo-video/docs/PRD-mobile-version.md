# PRD: 모바일 디바이스용 프로모션 영상

## 개요

기존 4K (3840x2160, 16:9) 프로모션 영상을 모바일 디바이스에 최적화된 버전으로 제작

## 목표

- 인스타그램 릴스, 틱톡, 유튜브 쇼츠 등 모바일 플랫폼에 최적화
- 세로 화면에서 가독성 확보
- 기존 콘텐츠 재활용으로 효율적 제작

---

## v1.1 개선사항 (2025-01-25)

### 문제점

4K 버전의 "텍스트 + 모바일 화면 동시 배치" 레이아웃은 세로 영상(9:16)에서:
- 모바일 화면이 너무 작게 보임
- 텍스트와 화면이 겹쳐 가독성 저하
- 시청자가 내용을 파악하기 어려움

### 해결 방안: 순차 등장 패턴

**기존 (4K)**
```
[텍스트] ←→ [모바일 화면]  (동시 배치)
```

**개선 (모바일 9:16)**
```
Phase 1: [텍스트] 등장 → 잠시 유지
Phase 2: [텍스트] 페이드아웃
Phase 3: [모바일 화면] 크게 등장 (화면 거의 꽉 참)
```

### 적용 대상 씬

| 씬 | 변경 내용 |
|----|----------|
| adToLanding | 텍스트+아이콘 → 페이드아웃 → 스마트폰 크게 |
| customerFlow | 텍스트+플로우 → 페이드아웃 → 스마트폰 크게 |
| realtimeAlert | 텍스트+흐름 → 페이드아웃 → 텔레그램 카드 크게 |
| portalLeads | 텍스트 → 페이드아웃 → 포털 화면 크게 |
| portalStats | 텍스트 → 페이드아웃 → 통계 화면 크게 |

### 모바일 화면 크기 기준

- 기존: scale 1.3~1.5 (화면의 일부)
- 개선: scale 2.2~2.5 (화면 거의 꽉 참, 좌우 여백 최소화)

### 애니메이션 타이밍 (예시)

```
0.0s - 텍스트 등장
0.8s - 텍스트 유지
1.3s - 텍스트 페이드아웃 시작
1.6s - 모바일 화면 등장 시작
2.0s - 모바일 화면 완전 등장
3.5s - 키포인트 등장
4.5s - 씬 종료
```

## 구현 방식

### 방식 1: 별도 프로젝트 생성 (권장)
```
promo-video/
├── src/project.ts          # 4K 버전 (기존)
├── src/project-mobile.ts   # 모바일 버전 (신규)
└── src/scenes/
    ├── opening.tsx         # 공유
    └── ...
```

**장점**: 기존 버전 유지, 독립적 수정 가능
**단점**: 일부 코드 중복

### 방식 2: 반응형 스타일 변수
```typescript
const MOBILE = { width: 1080, height: 1920 };
const scale = MOBILE.width / 3840; // 0.28125
```

모든 수치에 scale 적용
